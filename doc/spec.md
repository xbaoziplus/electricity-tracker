# 电费记账本 · 设计规格（三版本共同输入）

## 一、产品定义

「电费记账本」是一个**纯本地、可离线**的手机 Web App（最终形态 PWA），帮助个人/家庭：
1. 记录电表读数 → 自动算出每期用电量
2. 记录缴费金额 → 统计电费支出、估算余额
3. 查看用电趋势 → 近 12 个月用电量/费用图表
4. 阶梯电价计算 → 按年度累计用量拆档估算电费

无后端、无账号、无网络依赖，数据全部存 localStorage。中文界面，人民币计价。

## 二、目标用户与场景

- 家庭用户，每月（或不定期）抄一次自家电表；预付费电表用户想知道余额还能撑多久
- **场景 A**：站在楼道电表箱前，单手快速录入读数 → 大数字输入、大按钮、10 秒完成
- **场景 B**：晚上在沙发上翻趋势图，看这个月电用得凶不凶、阶梯爬到第几档
- 气质：**精密仪表**，不是营销页。数字是绝对主角（大号、tabular-nums、等宽对齐）

## 三、信息架构（底部 4 tab）

### Tab 1 总览 #overview
高密度数据工具，本屏至少呈现（这是产品差异化信息，不是装饰）：
- 本月用电量（大数字主角）+ 环比上月 ±%
- 日均用电（基于最近抄表区间）
- 当前阶梯档位 + 现行电价（如「第二档 · 0.5383 元/度」）
- 年度累计用电量 + 档位进度条（距下一档还剩 X 度）
- 本月估算电费
- 余额估算 = 累计缴费 − 累计估算电费（标注「估算」）
- 距上次抄表 N 天（超过 35 天给温和提醒）
- 两个快捷入口按钮：「记一笔读数」「记一笔缴费」（弹出与记录页相同的表单）

### Tab 2 记录 #records
- 顶部分段切换：电表读数 / 缴费记录
- 读数列表每行：日期、表读数、较上次 +Δ 度、间隔天数、该段估算电费
- 缴费列表每行：日期、金额、备注
- 新增/编辑用底部弹层（bottom sheet）：日期（默认今天）+ 数值大输入框 + 备注；可删除
- 读数输入校验：新读数小于上一次读数时警告（电表不会倒走，换表除外——允许强制保存）

### Tab 3 趋势 #trends
- 近 12 个月柱状图，**纯内联 SVG 手绘**（禁第三方图表库）
- 顶部切换：用电量（度）/ 费用（元）
- 点击柱子 → 显示该月明细（月份、度数、估算费用、日均）
- 图下一行统计：12 个月总用电 / 总费用 / 月均
- 不足 12 个月数据时只画有数据的月份，空月画淡色占位刻度

### Tab 4 设置 #settings
- 电价模式：单一电价（一个 元/度 输入）/ 阶梯电价（默认开）
- 阶梯表可编辑，默认国网典型年阶梯：
  - 第一档 ≤2160 度：0.4883 元/度
  - 第二档 2161–4800 度：0.5383 元/度
  - 第三档 >4800 度：0.7883 元/度
- 阶梯周期：按年（默认）/ 按月
- 数据管理：导出 JSON（下载文件）、导入 JSON、载入示例数据、清空全部数据（二次确认）
- 关于：一句话说明「数据仅保存在本机浏览器中」

## 四、数据模型（localStorage key = `elec.v1`）

```json
{

    "pricingMode": "tiered",
    "flatPrice": 0.55,
    "tiers": [
      { "limit": 2160, "price": 0.4883 },
      { "limit": 4800, "price": 0.5383 },
      { "limit": null, "price": 0.7883 }
    ],
    "tierPeriod": "year"
  },
  "readings": [ { "id": "r1", "date": "2026-06-28", "value": 12345.6, "note": "" } ],
  "payments": [ { "id": "p1", "date": "2026-06-20", "amount": 300, "note": "网上国网充值" } ]
}
```

## 五、核心计算规则（必须一致实现）

1. **用电量**：按日期排序的相邻两次读数之差 = 该区间用电量；除以间隔天数 = 日均
2. **月度归集**：把每个读数区间的用电量按天数均摊到自然日，再按自然月汇总 → 得每月 kWh（处理跨月抄表）
3. **阶梯拆分**：tierPeriod=year 时，每年 1 月 1 日累计清零；某月电费 = 将该月 kWh 叠加到「本年度此前累计」上，跨档部分分段计价求和。tierPeriod=month 时每月清零
4. **本月估算电费** = 上述规则算出的当月成本；**余额估算** = Σ缴费 − Σ全部月份估算成本
5. 单一电价模式：kWh × flatPrice
6. 读数不足 2 条时，总览显示引导空状态（「再记一次读数就能看到用电量了」），不编造数字

## 六、示例数据（?demo=1 自动载入，设置页也有按钮）

- 覆盖 2025-05-01 至 2026-07-01 共 15 次读数，起始表底 08210.5
- 月用量有真实季节感：春秋 180–240 度，夏季 7/8 月空调 380–430 度，冬季 1/2 月 320–360 度
- 缴费 8 笔，200–500 元不等，日期穿插其间，备注如「网上国网充值」「物业代收」
- 示例数据要在界面上可辨识（如设置页显示「当前为示例数据」），一键清空即可重来

## 七、技术硬约束

- **单文件 HTML + vanilla JS**，禁 React/Vue/任何 CDN 运行时、禁图表库；图标一律内联 SVG 线性图标（stroke 1.5–2px），**禁 emoji 图标**
- Google Fonts 允许引（最终版会用 Service Worker 缓存），但必须写系统字体兜底栈；中文正文用系统栈（PingFang SC / Microsoft YaHei 兜底）
- 移动优先：设计视口 390×844；`<meta name="viewport">`、`theme-color`、safe-area-inset 处理；触控目标 ≥44px
- URL hash 路由：#overview / #records / #trends / #settings 直达对应 tab；`?demo=1` 自动载入示例数据
- 所有数字展示用 `font-variant-numeric: tabular-nums`；金额保留 2 位小数，电量 1 位
- 双击文件（file://）即可打开运行，无需 server
- 交互必须真实可用：增删改、tab 切换、图表点击、设置修改全部生效并持久化

## 八、反 slop 红线

- 无紫色渐变、无 emoji 图标、无「圆角卡片+左彩色边条」偷懒组合
- 不编造装饰性 stats；每个数字都来自真实计算
- 空状态诚实设计（引导文案 + 主操作按钮），不放假图表
- 这是数据工具 → 走**高密度型**信息密度：总览屏 ≥3 处有内容的差异化信息

## 九、输出要求

- 输出到 `D:\tmp\object\electricity-tracker\design-demos\<版本名>.html`
- HTML 文件顶部用注释写清：本版设计逻辑、风格来源、关键 assumptions
- `document.title` 和 `theme-color` 按各自配色设置
- 不加水印（非动画产出）
